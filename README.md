
# お好み焼き屋さん 公式ホームページ
私がアルバイトをしている実際の店舗（お好み焼き・もんじゃ焼き店「ごきらく亭」）のオーナーからヒアリングを行い、実用性を重視して制作した公式ホームページです。

## 🔗 サイトURL
https://drew-works.github.io/okonomiyaki-site/HP/

## ✨ 本プロジェクトにおけるこだわり・開発プロセス

### 1. 徹底したユーザー目線のUX向上と課題解決
- **スマートフォンでの見やすさの追求：** 
  パソコンでは綺麗に見えても、スマートフォンで見るとキャッチコピーが画面幅のせいで変なところで改行されてしまう問題がありました。お店の第一印象である「アツアツ感」や魅力を損なわないよう、画面幅に合わせて文字が綺麗に収まるコード（`white-space: nowrap` と独自の改行コントロール）を実装し、スマホでの見やすさを徹底的に整えました。
- **アルバイト経験とヒアリングに基づく店舗課題の解決：** 
  実際にお店で働くなかで、週末の混雑時にお客様からよく電話でお問い合わせがある内容（服に匂いがつくか、どんな決済方法があるかなど）を分析しました。事前のお客様の不安を解消するために「よくあるご質問（Q&A）」ページを新設し、さらにスマホからワンタップでスムーズに電話予約ができるボタンを設置することで、お客様にとってもお店にとっても優しいサイト設計にしました。

### 2. 「AIとの共創」による開発速度の最大化と拡張性の担保
- **AIを開発パートナーとした設計の言語化：**
  これまで公式ホームページが存在しなかったため、オーナーの「ごきらく亭っぽくしてほしい！」という抽象的な要望に対し、まずは「一目見てお店について知りたい情報がすぐにわかるシンプルなサイトにする」という方針を定義しました。その上で、情報の優先順位（トプ画のインパクト、おしながき、Q&A、電話予約）や、下町の鉄板焼き屋の温かみを表現する配色設計についてAIと考え、最適なデータ構造やマークアップへと落とし込む協働開発を実践しました。
- **拡張性と保守性を意識したコード設計：**
  公開後、オーナーからのメニューについての細かい仕様変更の要望に対し、CSSのクラス名とHTMLのカード構造をシンプルに紐付ける設計をAIと共創。これにより、今後のメニュー変更にも即座に対応できるようにしました。また、価格改定に伴い、スマートフォンの限られた画面幅でも視認性を保てるよう、税込表記を薄くコントロールする専用スタイル（`tax-included`）への仕様改善を行いました。

### 3. 継続的改善
- **迅速な仕様変更への対応：**
  「作って終わり」ではなく、公開後も店舗オーナーからの要望を迅速に反映させるため、Gitを用いた確実なバージョン管理と、継続的な機能改善のサイクルを回し続けています。
  
## 🛠️ 使用技術

- CSS3（Flexbox, Grid Layout, メディアクエリによるレスポンシブデザイン）
- Git / GitHub
- AIツールを活用した協働設計
